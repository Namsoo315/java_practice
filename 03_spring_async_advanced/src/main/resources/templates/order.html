    서킷 브레이커 데모 안내
    

        이 주문 화면은 Resilience4j 서킷 브레이커 동작을 테스트하기 위한 데모이다.
    

    
    
        
  • 
            상품 ID 1 : 장애가 없는 정상 상품이다. 여러 번 주문을 보내도 항상 /mock-inventory 로 실제 재고 조회 요청이 간다.
        
    • 
        
  • 
            상품 ID 5 이상 (예: 6, 7) : 약 80% 확률로 실패하는 일시 장애 시나리오이다.
            
    
            sliding-window-size=5, failure-rate-threshold=60 이므로,
            최근 5번 호출 중 3번 이상 실패하면 서킷이 Open 상태가 되고,
            그 이후 요청부터는 재고 서비스로 요청을 보내지 않고 fallbackStock() 이 바로 호출된다.
        
    • 
        
  • 
            상품 ID 9 : 항상 장애가 발생하는 상품이다. 몇 번만 연속해서 주문을 보내면
            실패율이 60%를 넘어서 서킷이 Open 상태가 되고, 이후에는 재고 서비스로 요청을 보내지 않는다.
        
    • 
        
  • 
            서킷이 Open 상태가 되면 wait-duration-in-open-state=10초 동안은 모든 호출이 바로 차단되고
            fallback 응답만 내려간다. 10초가 지나면 Half-Open 상태로 전환되어 다시 일부 요청을 실제 재고 서비스로 보내고,
            성공 비율이 충분하면 자동으로 Closed 상태로 복구된다.
        
    • 
    

    

        ※ 상품 9로 여러 번 주문을 보내면서, 일정 횟수 이후부터는 재고 서비스 로그가 찍히지 않고
        fallback 로그만 찍히는 시점을 확인하면 서킷 브레이커 동작을 쉽게 관찰할 수 있다.
        

        이후 10초 정도 기다렸다가 다시 요청을 보내면, 서킷이 Half-Open → Closed 로 복구되는 과정도 확인할 수 있다.
